Le tri par insertion est le tri le plus connu. C'est celui que nous utilisons intuitivement quand nous devont trier une liste d'objets, par exemple quand nous jouons aux cartes.

Principe

Quand on considère une collection $l$ de $n$ éléments, dont les $i - 1$ premiers sont triés.
Placer le $i^{ème}$ élément, consiste à le déplacer dans la sous-collection triée jusqu'à sa "bonne" place.

La "bonne" place est l'indice $j$ où :

  • tous les éléments de $l[0:j]$ sont $\lt$ à l'élément $i$ et,
  • tous les éléments de $l[j + 1:i - 1]$ sont $\gt$ à l'élément $i$.

Illustration

En fonction du principe énoncé ci-dessus, écrire l'algorithme de tri par insertion.

Terminaison

Est-ce que l'algorithme suivant termine ? 

Algorithme : "Tri par insertion"
Entrée : l : une collection de n éléments
Sortie : Aucune
Effet de bord : l est triée
  Début
  Pour i de 1 à n - 1
    # mémoriser l'élément en cours
    element ← l[i]

    # décaler vers la droite les éléments de l[0]..l[i-1]
		# qui sont plus grands que element en partant de l[i-1]
    j ← i
    Tant que j > 0 et l[j - 1] > element
      l[j] ← l[j - 1]
      j ← j - 1
    FinTantQue
    # placer element dans le "trou" laissé par le décalage
    l[j] ← element
  FinPour
  Fin

L'algorithme "Tri par insertion" contient une boucle Tant que, il s’agit donc de vérifier qu’elle se termine.
La variable $j$ démarre de $i$ et diminue d’une unité à chaque itération.
En $n - 1$ itérations, $j$ prendra la valeur 0, ce qui permet (si ce n’est pas encore fait) de sortir de la boucle.

$j$ est un variant de boucle, i.e une quantité :

  • entière,
  • positive,
  • strictement décroissante.

En conclusion, l'algorithme "Tri par insertion" se termine.
